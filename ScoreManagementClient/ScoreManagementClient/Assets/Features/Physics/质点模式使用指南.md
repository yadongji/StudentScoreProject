# 质点模式使用指南

## 一、什么是质点？

### 物理学中的质点定义
**质点**是指：
- 没有大小和形状的理想物体
- 所质量集中在一个几何点上
- 不考虑转动惯量和旋转
- 所受力都作用在质心

### 在物理实验中的作用
在传送带实验中使用质点模型可以：
- ✅ 简化物理计算（只考虑平移）
- ✅ 避免形状和尺寸对实验的干扰
- ✅ 更好地验证基本物理公式（如 mgsinα = μmgcosα）
- ✅ 符合理论物理教材的简化模型

## 二、Unity 中的质点模式实现

### 1. PhysicsObject 组件参数

在 Inspector 中可以看到以下质点相关参数：

```
[质点模式]
├─ Is Particle: [勾选框] 启用/禁用质点模式
├─ Particle Collider Size: 0.01 m 质点碰撞器大小
└─ Particle Visual Size: 0.1 m 质点可视化大小
```

### 2. 启用质点模式的方法

#### 方法 A：在 Inspector 中勾选
1. 选择带有 PhysicsObject 组件的对象
2. 在 Inspector 中勾选 **Is Particle**
3. 物体立即转换为质点模式

#### 方法 B：使用右键菜单
1. 在 Hierarchy 中右键点击对象
2. 选择 PhysicsObject 组件
3. 点击以下任一选项：
   - **切换质点模式**
   - **启用质点模式**
   - **禁用质点模式**

### 3. 质点模式的效果

#### 启用质点模式后：
```
✅ 锁定旋转（Rigidbody.FreezeRotation）
✅ 碰撞器缩小到极小尺寸（0.01m）
✅ 创建小球可视化（Particle Visual）
✅ 隐藏原始模型
✅ 所受力作用在质心
✅ 忽略扭矩和角速度
```

#### 禁用质点模式后：
```
✅ 恢复旋转自由度
✅ 恢复原始碰撞器尺寸
✅ 销毁小球可视化
✅ 显示原始模型
✅ 恢复完整的物理模拟
```

## 三、质点模式在实验中的应用

### 应用场景 1：验证平衡条件 mgsinα = μmgcosα

#### 设置
```
1. 创建传送带对象，添加 BeltConveyor 组件
2. 设置倾角：例如 30°
3. 创建滑块对象：
   - 添加 PhysicsObject 组件
   - 勾选 Is Particle（质点模式）
   - 设置 Mass = 1 kg
4. 添加 DataLogger 记录数据
5. 添加 EnergyCalculator 计算能量
```

#### 验证过程
```
1. 运行场景
2. 物体表现为质点（小球）在传送带上
3. 调节传送带倾角
4. 观察数据：
   - 当角度较小时，质点静止
   - 当角度超过临界值，质点开始下滑
5. 使用 DataLogger 导出数据
6. 验证理论计算与实验数据是否吻合
```

### 应用场景 2：研究摩擦系数的影响

#### 设置
```
1. 质点模式已启用
2. 固定倾角：例如 30°
3. 调节物理材质的摩擦系数
   - Dynamic Friction = μ
   - Static Friction = μ
4. 观察质点运动状态
```

#### 观察点
```
✓ 质点不受形状影响，纯粹由摩擦力决定
✓ 可以精确测量临界摩擦系数
✓ 数据更符合理论公式
✓ 避免物体边缘、角等因素的干扰
```

### 应用场景 3：计算临界角度

#### 目标
给定摩擦系数 μ，求临界角度 α = arctan(μ)

#### 实验步骤
```
1. 设置摩擦系数 μ（例如 0.577）
2. 启用质点模式
3. 缓慢增加传送带倾角（0° → 90°）
4. 观察质点开始下滑的角度
5. 记录临界角度 α临界
6. 理论验证：tan(α临界) ≈ μ
```

#### 预期结果
```
当 μ = 0.577 时：
理论临界角度 = arctan(0.577) ≈ 30°
实验测量角度 ≈ 30°（理想情况下）
```

## 五、代码接口

### 检查是否为质点模式
```csharp
PhysicsObject obj = GetComponent<PhysicsObject>();

if (obj.IsParticle)
{
    Debug.Log("这是质点模式");
}
else
{
    Debug.Log("这是普通模式");
}
```

### 动态切换质点模式
```csharp
// 方法 1：通过属性
obj.IsParticle = true;

// 方法 2：通过方法
obj.EnableParticleMode();    // 启用
obj.DisableParticleMode();  // 禁用
obj.ToggleParticleMode();     // 切换
```

### 质点模式下的物理特性
```csharp
// 质点模式下，以下操作被忽略：
obj.ApplyTorque(Vector3.zero);           // ❌ 被忽略
obj.SetAngularVelocity(Vector3.zero);     // ❌ 被忽略
obj.GetAngularVelocity();               // ❌ 返回零

// 质点模式下，以下操作正常工作：
obj.ApplyForce(Vector3.forward * 10f);   // ✅ 正常
obj.SetVelocity(Vector3.forward * 2f);   // ✅ 正常
obj.GetVelocity();                       // ✅ 正常
obj.GetKineticEnergy();                  // ✅ 正常（1/2mv²）
```

## 六、实验建议

### 何时使用质点模式

#### 推荐使用的情况：
- ✅ 验证基础物理公式（如 mgsinα = μmgcosα）
- ✅ 理论分析实验
- ✅ 需要高精度数据
- ✅ 教材中使用质点模型

#### 不推荐使用的情况：
- ❌ 研究形状对运动的影响
- ❌ 研究旋转和转动惯量
- ❌ 需要可视化真实物体
- ❌ 实际工程仿真

### 参数设置建议

#### Particle Collider Size（质点碰撞器大小）
```
推荐值：0.01 m（1 cm）

设置原则：
- 太小（< 0.005）：Unity 物理引擎可能不稳定
- 太大（> 0.05）：失去质点意义
- 推荐范围：0.005 - 0.02 m
```

#### Particle Visual Size（质点可视化大小）
```
推荐值：0.1 m（10 cm）

设置原则：
- 足够大，便于观察
- 不要太大，避免视觉干扰
- 推荐范围：0.05 - 0.2 m
```

## 七、常见问题

Q1: 为什么质点要锁定旋转？
A: 质点定义为没有转动惯量的理想物体。
   锁定旋转可以确保质点只做平移运动，
   不受力矩影响，符合质点的物理定义。

Q2: 质点的碰撞器为什么不是零？
A: Unity 物理引擎无法处理零体积的碰撞器。
   设置为极小值（如 0.01 m）可以实现近似零体积的效果，
   同时保持物理引擎的稳定性。

Q3: 质点模式会影响质量吗？
A: 不会。质量由 Mass 属性独立设置。
   无论是否启用质点模式，质量都保持不变。

Q4: 质点模式下的动能计算是否改变？
A: 不改变。质点的动能依然是 Ek = ½mv²。
   唯一区别是质点没有转动动能。

Q5: 可以在运行时切换质点模式吗？
A: 可以！通过 IsParticle 属性或右键菜单都可以动态切换。
   切换会立即生效，无需重启场景。

### 实验：验证临界摩擦系数公式 μ = tanα

#### 场景搭建
```
1. 创建传送带
   - GameObject: "BeltConveyor"
   - 组件: BeltConveyor
   - Incline Angle: 0°（从 0° 开始）

2. 创建质点物体
   - GameObject: "Particle"
   - 添加 PhysicsObject 组件
   - Mass: 1 kg
   - Is Particle: ☑️ 勾选
   - Particle Collider Size: 0.01
   - Particle Visual Size: 0.1

3. 添加数据采集
   - 在 Particle 上添加 DataLogger
   - Target Object: 拖入 Particle 对象

4. 添加能量计算器
   - 在 Particle 上添加 EnergyCalculator
   - Target Object: 拖入 Particle 对象
```

#### 实验步骤
```

1. 设置物理材质：Dynamic Friction = 0.3
2. 运行场景

4. 观察质点开始下滑的角度
5. 记录：α临界 ≈ ?

4. 观察并记录临界角度

1. 设置物理材质：Dynamic Friction = 1.0

4. 观察并记录临界角度
```
